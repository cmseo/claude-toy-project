# Tennis Playbook AI 연동 — spec.md

## 개요

플레이북 생성을 시뮬레이션(문장 분리·랜덤 선별)에서 실제 AI(Google Gemini)로
교체하여, 전체 다이어리를 분석한 맥락 있는 행동 지침 체크리스트를 제공한다.

## 범위

### 포함
- 기록 저장(추가·편집·삭제) 시 AI가 전체 다이어리를 기반으로 플레이북 5개 항목을 새로 생성
- 플레이북 생성 중 로딩 상태: 스피너 + "플레이북 생성 중..." 텍스트
- AI 실패 시 에러 토스트 + 기존 플레이북 유지
- 플레이북 탭에 수동 "새로고침" 버튼 — 언제든 재생성 가능
- 환경변수 기반 API 키 관리 (Vercel 배포 호환)

### 제외
- 플레이북 항목 출처(날짜) 표시 — 현재는 텍스트만 표시하여 단순성 유지
- 스트리밍 응답 — 5개 항목 생성은 충분히 빠르므로 불필요
- 사용자별 프롬프트 커스터마이징 — 개인 프로젝트 범위를 넘음
- fallback 시뮬레이션 — AI 실패 시 기존 플레이북 유지로 충분, 시뮬레이션 재호출 불필요

## 시나리오

### 1. 기록 저장 후 AI 플레이북 생성 (성공)

- **Given** 기록이 1개 이상 저장된 상태에서 새 기록을 저장한다
- **When** 저장 완료 직후
- **Then** 플레이북 탭에 로딩 스피너와 "플레이북 생성 중..." 텍스트가 표시되고, AI 응답 후 새 플레이북 5개 항목으로 전체 교체되며, 마지막 갱신 시각이 업데이트된다

성공 기준:
- [ ] 기록 저장 직후 플레이북 영역에 스피너와 "플레이북 생성 중..." 텍스트가 표시된다
- [ ] AI 응답 완료 후 플레이북에 5개 항목이 표시된다
- [ ] 이전 플레이북 항목은 모두 사라지고 새 항목으로 교체된다
- [ ] "마지막 갱신: [날짜 시각]"이 업데이트된다
- [ ] 생성된 항목은 한국어 행동 지침 형태이다

---

### 2. AI 플레이북 생성 실패

- **Given** 기록을 저장했으나 AI 서비스가 응답하지 않는다 (네트워크 오류, API 한도 초과 등)
- **When** AI 호출이 실패한다
- **Then** 에러 토스트가 표시되고, 기존 플레이북이 그대로 유지된다

성공 기준:
- [ ] AI 호출 실패 시 "플레이북 생성에 실패했습니다" 토스트가 표시된다
- [ ] 기존 플레이북 항목이 그대로 유지된다 (빈 상태로 바뀌지 않음)
- [ ] "마지막 갱신" 시각이 변경되지 않는다

---

### 3. 수동 새로고침

- **Given** 플레이북 탭이 열려 있고 기록이 1개 이상 존재한다
- **When** "새로고침" 버튼을 탭한다
- **Then** AI가 현재 전체 다이어리 기반으로 플레이북을 새로 생성한다

성공 기준:
- [ ] 플레이북 탭에 "새로고침" 버튼이 표시된다
- [ ] 버튼 탭 시 로딩 스피너와 "플레이북 생성 중..." 텍스트가 표시된다
- [ ] 생성 완료 후 새 플레이북으로 교체된다
- [ ] 기록이 0개일 때는 "새로고침" 버튼이 표시되지 않는다

---

### 4. 첫 기록 저장 시 플레이북 최초 생성

- **Given** 아직 저장된 기록이 없는 상태
- **When** 첫 번째 기록을 저장한다
- **Then** AI가 해당 기록을 기반으로 플레이북을 생성하고, 빈 상태가 플레이북 항목으로 교체된다

성공 기준:
- [ ] 첫 기록 저장 후 플레이북 탭에서 빈 상태 문구가 사라진다
- [ ] AI가 생성한 플레이북 항목이 표시된다
- [ ] FAB 버블 가이드가 사라진다

---

## 불변 규칙

- **기록 저장 독립성**: 기록 저장은 AI 호출과 독립적으로 항상 성공한다. AI 실패가 기록 저장을 롤백하지 않는다
- **플레이북 무손실**: AI 실패 시 기존 플레이북이 손실되지 않는다. 새 플레이북으로 교체는 AI 성공 시에만 발생한다
- **응답 시간**: 플레이북 생성은 기록 저장 후 10초 이내에 완료된다 (정상 네트워크 환경)
- **기존 동작 유지**: 기록 추가·편집·삭제, 히스토리, 시간 요약, 다크 모드 등 기존 기능은 변경 없이 동작한다
