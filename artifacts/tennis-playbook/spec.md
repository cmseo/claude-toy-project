# Tennis Playbook — spec.md

## 개요

테니스 레슨·경기의 느낀점을 다이어리로 기록하고, 기록을 기반으로 자동 갱신되는
경기 전 플레이북 체크리스트를 통해 배움이 실제 행동 변화로 이어지게 한다.

## 범위

### 포함
- 레슨/경기 기록: 날짜, 유형, 소요 시간(분), 느낀점 (자유 텍스트)
- 경기 기록에만 스코어(자유 텍스트)·승패 추가
- 기록 편집·삭제
- 플레이북: 새 기록 저장마다 다이어리 기반으로 자동 전체 교체 (읽기 전용)
- 히스토리: 날짜 역순 기록 목록 + 기간별 총 소요 시간 요약 (월 기본 / 주·년 탭 전환, 시간/분 형식)
- 네비게이션: 하단 2탭 (플레이북 / 히스토리) + 중앙 FAB (기록 추가)
- FAB 버블 가이드: 기록이 없을 때 FAB 위에 말풍선으로 첫 기록 유도
- 다크 모드: 시스템 설정 연동 자동 전환
- 랜딩 페이지: 앱 소개 + 시작하기 버튼 (최초 진입 시)
- Vercel 배포

### 제외
- 플레이북 수동 추가·편집·삭제 — 자동 관리하며 사용자가 개입하지 않음
- 실제 AI API 연동 — 이번 사이클은 시뮬레이션으로 검증, 다음 사이클에서 확장
- 태그·검색·필터 — 핵심 가정 검증에 불필요
- 자유 연습 유형 — 레슨/경기 두 유형으로 핵심 가정 검증 충분
- 영상·사진 첨부 — 구현 비용 대비 핵심 가치와 거리 있음
- 로그인·클라우드 동기화 — 개인 실습 프로젝트 범위를 넘음
- JSON export — 로컬 전용으로 복잡도 최소화, 검증 후 재검토

## 시나리오

### 1. 레슨 기록 추가

- **Given** 기록 추가 탭이 열려 있고, 유형 "레슨"이 선택된 상태
- **When** 날짜·소요 시간을 입력하고 (느낀점은 선택 입력) "저장" 버튼을 탭한다
- **Then** 히스토리 탭 최상단에 새 레슨 카드가 나타나고, 플레이북이 자동 갱신된다

성공 기준:
- [ ] 날짜가 비어 있으면 "날짜를 입력하세요" 오류 메시지가 필드 아래에 표시된다
- [ ] 소요 시간이 비어 있거나 0 이하이면 "소요 시간을 입력하세요" 오류 메시지가 표시된다
- [ ] 저장 성공 시 입력 폼이 초기화되고, 날짜는 오늘로 기본값이 설정된다
- [ ] 히스토리 탭에 방금 입력한 날짜·유형·소요 시간·느낀점이 표시된다

---

### 2. 경기 기록 추가

- **Given** 기록 추가 탭에서 유형 "경기"를 선택한 상태
- **When** 날짜·소요 시간·스코어·승패를 입력하고 저장한다
- **Then** 히스토리에 스코어와 승패가 표시된 경기 카드가 나타나고, 플레이북이 갱신된다

성공 기준:
- [ ] 유형이 "경기"일 때만 스코어·승패 필드가 화면에 표시된다
- [ ] 유형이 "레슨"으로 바뀌면 스코어·승패 필드가 사라진다
- [ ] 히스토리의 경기 카드에 스코어와 승/패가 표시된다

---

### 3. 플레이북 자동 갱신

- **Given** 기록이 1개 이상 저장된 상태에서 새 기록을 저장한다
- **When** 저장 직후
- **Then** 플레이북 탭의 체크리스트가 다이어리 기반으로 선별된 새 항목들로 전체 교체되고, 마지막 갱신 시각이 표시된다

성공 기준:
- [ ] 저장 직후 플레이북에 항목 목록이 표시된다
- [ ] 이전 플레이북 항목은 모두 사라지고 새 항목으로 교체된다
- [ ] 플레이북 화면에 "마지막 갱신: [날짜 시각]"이 표시된다

---

### 4. 플레이북 빈 상태 + FAB 버블 가이드 (기록 없음)

- **Given** 아직 저장된 기록이 없다
- **When** 플레이북 탭을 연다
- **Then** 빈 상태 안내 문구가 표시되고, FAB 위에 말풍선이 나타나 첫 기록을 유도한다

성공 기준:
- [ ] 기록이 0개일 때 "아직 기록이 없어요." 문구가 표시된다
- [ ] FAB 위에 "첫 기록을 남겨보세요!" 말풍선이 표시된다
- [ ] FAB를 탭하면 기록 추가 화면으로 이동한다

---

### 5. 기록 편집

- **Given** 히스토리에 기록 카드가 표시된 상태
- **When** 카드의 편집 버튼을 탭한다
- **Then** 해당 기록의 내용이 채워진 편집 폼이 열린다

성공 기준:
- [ ] 편집 폼에 기존 날짜·유형·소요 시간·느낀점이 채워져 있다
- [ ] 편집 후 저장하면 히스토리 카드에 수정 내용이 반영된다
- [ ] 편집 저장 시에도 플레이북이 갱신된다

---

### 6. 기록 삭제

- **Given** 히스토리에 기록 카드가 표시된 상태
- **When** 카드의 삭제 버튼을 탭하고 확인 다이얼로그에서 확인한다
- **Then** 해당 카드가 히스토리에서 제거되고 플레이북이 갱신된다

성공 기준:
- [ ] 삭제 버튼 탭 시 "삭제하시겠습니까?" 확인 다이얼로그가 표시된다
- [ ] 확인 후 해당 카드가 히스토리에서 사라진다
- [ ] 삭제 후 플레이북이 남은 기록 기반으로 갱신된다
- [ ] 모든 기록이 삭제되면 플레이북에 빈 상태 메시지가 표시된다

---

### 7. 히스토리 시간 요약

- **Given** 히스토리 탭을 연다
- **When** 상단 요약 카드를 본다
- **Then** 이번 달 총 테니스 소요 시간(분)이 표시되고, 탭으로 주·년·전체 합계로 전환할 수 있다

성공 기준:
- [ ] 기본으로 이번 달 기록의 소요 시간 합계가 표시된다
- [ ] "주" 탭을 누르면 이번 주 합계로, "년" 탭을 누르면 올해 합계로 바뀐다
- [ ] "전체" 탭을 누르면 첫 기록 날짜부터의 누적 합계가 "YY년 MM월 DD일부터 X시간 Y분" 형식으로 표시된다
- [ ] 해당 기간에 기록이 없으면 "0분"이 표시된다

---

## 불변 규칙

- **데이터 지속성**: 앱을 닫았다 열어도 기록과 플레이북이 그대로 유지된다
- **갱신 순서**: 플레이북 갱신은 기록 저장 성공 후에만 발생한다. 기록 저장 실패 시 갱신이 발생하지 않는다
- **다크 모드**: 기기의 다크 모드 설정에 따라 모든 화면이 자동으로 전환된다
- **시간 표시**: 소요 시간은 분(分) 단위로 입력하되, 화면에 표시할 때는 1시간 이상이면 "X시간 Y분" 형식으로 변환한다 (예: 90분 → 1시간 30분, 60분 → 1시간)
