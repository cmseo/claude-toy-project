# Tennis Playbook

## Problem Statement
HMW — 테니스 훈련에서 얻은 인사이트가, 기록으로 쌓이는 데서 끝나지 않고
다음 경기의 실제 행동 변화로 연결되게 할 수 있을까?

## Recommended Direction
레슨/경기 직후 느낀점을 짧게 남기는 다이어리와, AI가 자동 생성하는
경기 전 플레이북 체크리스트를 조합한다. 사용자는 다이어리만 쓰고,
플레이북은 AI가 전체 기록을 검토하여 자동으로 갱신한다.

플레이북은 경기 전 30초 안에 읽을 수 있는 체크리스트다. 새 기록을
저장할 때마다 AI가 다이어리 전체를 읽고 전체 교체한다. 사용자가
직접 추가하거나 수정하지 않는다 — AI에게 완전히 위임한다.

데이터는 로컬(localStorage)에만 저장한다. 서버 없이 빠르게 만들 수 있고,
실습 범위에도 맞다. 기기 분실 위험은 감수하되, 검증 후 다음 사이클에서
export 기능을 고려한다.

## Key Assumptions to Validate
- [ ] 레슨/경기 직후 1분 안에 기록을 남길 수 있다
      → 검증: 직접 입력 플로우 사용
- [ ] AI가 생성한 플레이북을 경기 전에 실제로 열람한다
      → 검증: 플레이북 탭 열람 여부 (Must Be True)
- [ ] AI 생성 항목이 실제 경기에서 행동 변화로 이어진다
      → 검증: 다이어리에 "오늘 플레이북에서 본 것을 실천했다" 류의 기록이 남는지
- [ ] 주/월/년 시간 요약이 동기부여와 회고에 도움이 된다
      → 검증: 히스토리 요약 카드 반응

## Success Criteria (첫 사이클 종료 기준)
- 2주 안에 기록 10건 이상
- 플레이북을 경기 전 1회 이상 열람

## MVP Scope
- 랜딩 페이지: 앱 소개 + 시작하기 버튼
- 기록: 날짜, 유형(레슨/경기), 소요 시간(분), 느낀점 (단일 자유 텍스트)
- 경기 기록에만 스코어(자유 텍스트)·승패 추가
- 기록 저장 시 다이어리 전체를 기반으로 플레이북 자동 전체 교체 (시뮬레이션)
- 네비게이션: 하단 2탭 (플레이북 / 히스토리) + 중앙 FAB (기록 추가)
- FAB 버블 가이드: 기록 0개 상태에서 FAB 위 말풍선으로 첫 기록 유도
- 플레이북 탭 (기본 진입): 번호 리스트 체크리스트 (읽기 전용)
- 히스토리 탭: 날짜 역순 기록 목록 + 기간별 총 시간 (월 기본, 주·년 탭 전환, 시간/분 형식)
- 기록 편집·삭제 (삭제 시에도 플레이북 갱신)
- 다크 모드: 시스템 설정 연동 자동 전환
- 로컬 저장 (localStorage)

## Not Doing (and Why)
- 자유 연습 유형 — 데이터 모델 단순하게 유지. 레슨/경기로 핵심 가정 검증 충분
- 플레이북 수동 편집/추가 — 자동 갱신 구조. 사용자 개입 없음
- 실제 AI API 연동 (Anthropic) — 이번 사이클은 시뮬레이션으로 검증. 3시간 제약과 Vercel 배포를 고려해 다음 사이클에서 확장
- 태그/검색/필터 — 첫 사이클 핵심 가정 증명에 불필요
- 통계 차트/상대 분석 — 데이터 구조와 UI 비용이 커짐
- 클라우드 동기화 / JSON export — 로컬 전용으로 복잡도 최소화. 검증 후 재검토
- 영상/사진 첨부 — 핵심 가치와 거리 있음
- 로그인/멀티유저 — 개인 프로젝트 실습 범위를 넘음

## Open Questions
- 플레이북 항목 개수: AI가 몇 개를 생성할지 (고정 3-5개 vs 유동)
- 플레이북 항목 출처 표시: 어느 날짜 기록에서 도출됐는지 보여줄지
